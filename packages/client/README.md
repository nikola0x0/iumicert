# IU-MiCert Student/Verifier Portal

Public-facing Next.js web application for students and third-party verifiers to upload and verify academic credentials.

## Purpose

This portal enables:

- **Students**: Share and verify their academic receipts
- **Employers**: Verify student credentials independently
- **Institutions**: Verify credentials from other universities

## Live Demo

**Production URL**: [https://iu-micert.vercel.app](https://iu-micert.vercel.app)

## Features

### Receipt Verification

- Upload receipt JSON files
- View complete academic journey
- See individual course details with cryptographic proofs
- Verify blockchain anchoring status for each term
- Search students by ID

### Proof Display

- 32-byte IPA (Inner Product Argument) proofs for each course
- Term-by-term verification status
- Expandable course cards with grades and completion dates
- Visual indicators for verified/unverified terms

### Privacy-Preserving

- No data stored on server
- All verification happens client-side or via API calls
- Students control what they share (selective disclosure)

## Getting Started

### Prerequisites

- Node.js 18+
- Backend API running (see `packages/issuer/`)

### Installation

```bash
cd packages/client/iumicert-client
npm install
```

### Configuration

Create `.env.local`:

```bash
NEXT_PUBLIC_API_URL=http://localhost:8080
```

For production (Vercel):

```bash
NEXT_PUBLIC_API_URL=http://YOUR_BACKEND_IP:8080
```

### Development

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

### Build

```bash
npm run build
npm start
```

## How Verification Works

1. **Upload Receipt**: User uploads a receipt JSON file generated by the issuer
2. **Parse Data**: Extract student info, terms, courses, and cryptographic proofs
3. **Verify Proofs**: Send proofs to backend API for IPA verification
4. **Check Blockchain**: Verify term roots are published on Ethereum Sepolia
5. **Display Results**: Show verification status with visual indicators

## API Integration

The portal communicates with the backend API (packages/issuer/) for:

- `/api/verifier/receipt` - Verify complete receipt
- `/api/verifier/course` - Verify individual course proof
- `/api/blockchain/transaction/{hash}` - Check blockchain transaction status

See `packages/issuer/README.md` for API documentation.

## Security

- No sensitive data stored
- All cryptographic verification done server-side
- CORS configured for security
- No authentication required (public verification)

## Deployment

### Vercel (Recommended)

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel --prod
```

Add environment variable in Vercel dashboard:

- `NEXT_PUBLIC_API_URL` = Your backend API URL

### Docker

```bash
docker build -t iumicert-client .
docker run -p 3000:3000 -e NEXT_PUBLIC_API_URL=http://your-api iumicert-client
```
